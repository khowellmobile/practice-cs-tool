Directions

Learn to use Practice Report

Generate Report

Create custom reports effortlessly. Select from a curated list of available reports. Search and sort however you like.

Input Fields

  • Dates (required) - The date inputs follow the standard MM/DD/YYYY format. These represent the start and ending date for the requested report.
  • Date Preset (required) - The date preset allows the user to select a date range preset. Changing the dates manually will auto-select the custom preset.

Presets

  • Custom - A custom date range entered by the user.
  • All Time - A date range encomppasing all information up until the run report button is clicked.
  • Year to Date - Covers the first of teh year to the current date and time.
  • Last Year - Includes information from the 1st of January to the 31st of December of the previous year.

Report Functions

  • Sorting - Click on a column title to sort the report by that column.
  • Searching - Type in the search box to quickly find any specific information.
  • Pagination - The number of results per page can be changed using the drop down in the upper left hand corner
  • Switch Pages - The page selector can be found in the lower right hand corner

Purpose

See your previously run reports and quickly rerun them for access to commonly used reports.

History Listing

  • Each report that you run will have its parameters stored and listed here. Click on one to be taken to the generate report page with the paramters prefilled.

Fields

  • Each parameter required to run the report is listed. This includes date range preset, start date, end date, and the date the report was initially ran on.

Change Database

Seamlessly switch to a different database by entering all required fields. Monitor the connection status and stay informed about any potential issues that may arise during the process.

Input Fields

  • Engine (required) - Accepted engines are postgresql, mysql, sqlite', oracle, and mssql.
  • Name (required) - Only alphanumeric characters and underscores are allowed.
  • Host (optional) - Valid host names, domain names, and IPv4 addresses are allowed.
  • Driver (optional) - Can include alphanumeric characters, underscores, and spaces.
  • Username and Password - The connection will use Windows Authentication if these are left blank. Each database has the potential to have different Username and Password rules. Check your database information for allowed formats.

Connection Status

  • This will keep up updated about your connection. Any errors are displayed hear with common errors having recommendations on potential fixes.

Database History

  • Databases you have connected to in the past are listed here. Click on one to prefill the fields with your previously entered information. Usernames and Passwords are not saved.

Account Information

Review your account information easily. Add optional details to enhance your profile and update your information as needed.

Below is a listing of account fields and their required format.

  • Name (required) - Must be at least 2 characters long and can only include letters (A-Z, a-z), spaces, apostrophes ('), and hyphens (-).
  • Email (required) - Must follow the standard format. Allowed characters are letters, numbers, and special characters (., %, +, -). It must include an "@" symbol and a domain name with at least two letters after a period.
  • Phone Number (optional) - Must follow the standard format, allowing for an optional country code (e.g., +1) and can include spaces, dashes, or parentheses. It must contain at least 7 digits.
  • Company Name (optional) - Must be at least 2 characters long and can only include letters, numbers, spaces, and certain special characters (e.g., &, ., ',', '-').
  • Password (required) - Must be at least 8 characters long and include at least one special character (e.g., !@#$%^&*()_+={} :;<>,.?) and at least one digit.
